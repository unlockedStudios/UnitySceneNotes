# Repository Instructions

For any work in this repository, read `.github/copilot-instructions.md` first.

Then load only the instruction files that `.github/copilot-instructions.md` routes you to:

- Always load the core instruction files listed there.
- Load subsystem or workflow-specific files only when the task actually touches that area.
- Do not load every Markdown file under `.github/instructions/` by default.

Treat `.github/copilot-instructions.md` and the instruction files it selects as the canonical project instructions for this repository.

If guidance overlaps, prefer the more specific instruction file for the area you are working in.

Keep the actual project rules in the Copilot instruction files listed above so future updates are picked up automatically.
